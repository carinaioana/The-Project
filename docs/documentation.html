1. Introduction

1.1 Purpose

This documentation aims to provide insights into the app's functionality, features, and benefits, enabling users and developers to gain a thorough understanding of the app's capabilities and its value in exploring unemployment statistics in Romania.

1.2 Intended Audience and Reading Suggestions

For developers interested in the technical aspects of the Unemployment Statistics App, this documentation offers insights into the app's architecture, design principles, and data sources. Users interested in exploring unemployment statistics in Romania will find this documentation valuable in understanding how to navigate the Unemployment Statistics App effectively. It provides an overview of the app's user interface, available features, and data visualization options.

1.3 Product Scope

The Unemployment Statistics App is a comprehensive tool that provides accurate and up-to-date statistical information about unemployment in Romania. The app encompasses the following key features:

1.3.1 Interactive Map with County-Based Statistics: The app includes an interactive map that allows users to explore unemployment statistics based on different counties in Romania. Users can visualize the variations in unemployment rates across the country, identify areas with higher or lower levels of unemployment, and gain insights into regional trends. The map provides an intuitive and visually appealing way to understand the spatial distribution of unemployment.

1.3.2 Comparative Analysis by Demographic Factors: The app enables users to perform comparative analysis by demographic factors, including age, education, environment, and county. Users can select specific demographic variables and compare the corresponding unemployment rates. This feature helps users understand how unemployment affects different demographic groups, identify disparities, and uncover patterns or trends within specific categories.

1.3.3 Data Visualization and Insights: The app offers a variety of data visualization options to enhance understanding and interpretation of unemployment statistics. Users can access visually appealing charts, graphs, and maps that present data in a clear and concise manner. User-Friendly Interface and Customization: It offers intuitive navigation and interactive features that allow users to customize their data exploration. Users can filter data based on specific criteria, adjust visualizations to their preferences, and save or export selected statistics for further analysis or reporting purposes.

2.3 Operating Environment

The application will work on all operating systems available, such as Windows, Linux, macOS, on all web browsers and on mobile operating systems such as iOS, Android and Windows. A stable internet connection is needed for using the application.

2.4 Design and Implementation Constraints

The design and implementation of the Unemployment Statistics App involve several constraints, particularly in terms of the technologies used for the front-end and back-end development. The following constraints need to be considered:

  • Front-End:
    • HTML: The app's front-end will be developed using HTML, a widely supported markup language for structuring and presenting web content.
    • CSS + Responsive: Cascading Style Sheets (CSS) will be utilized to define the app's visual layout, styling, and presentation, ensuring a consistent and visually appealing user interface.
    • JavaScript: JavaScript will be employed for enabling dynamic interactions and enhancing user experience through interactivity and data manipulation.
  • Back-End:
    • Node.js: The back-end of the app will be implemented using Node.js, a JavaScript runtime environment that allows server-side execution of JavaScript code. Node.js offers scalability and non-blocking I/O operations.
    • PostgreSQL: PostgreSQL, a powerful open-source relational database management system, can be considered for storing and managing the app's data. PostgreSQL offers robust features, data integrity, and scalability, making it suitable for handling unemployment statistics.
    • JSON: JSON (JavaScript Object Notation) will be used as a lightweight data interchange format, facilitating efficient communication between the front-end and back-end components of the app.
    • Exports:
      • CSV: This feature allows users to download the data and use it in external applications or perform further analysis.
      • SVG: To enhance data visualization, the app can generate SVG (Scalable Vector Graphics) charts. SVG charts provide interactive and visually appealing representations of unemployment statistics.
      • PDF: The app may include the functionality to generate PDF (Portable Document Format) reports or summaries based on selected unemployment statistics. PDF reports provide a convenient way for users to present or share information.

4. System Features

4.1 Charts

  • 4.1.1 Description and Priority
    • Description: This feature enables users to visualize unemployment statistics using different types of charts based on selected filters. The available chart types include a pie chart for education filters, a horizontal bar chart for environment filters, and a bar chart for age filters. Users can also apply additional filters such as selecting a specific time frame (3, 6, or 12 months ago) and choosing a specific county in Romania. The charts are generated using the Chart.js library, providing interactive and visually appealing data representations.
    • Priority: High
  • 4.1.2 Stimulus/Response Sequences List
      • Stimulus:
    • User selects education/environment/age filters, time frame, and a specific county.
      • Response:
    • App retrieves unemployment data based on the selected education filters, time frame, and county.
    • App generates a pie chart using Chart.js to visualize the unemployment statistics for the selected filters.
    • User can hover over the chart slices/bars to see the corresponding label and data.
  • 4.1.3 Functional Requirements
    • The app should provide a user interface with options to select education filters, environment filters, and age filters.
    • Users should be able to select a specific time frame (3, 6, or 12 months ago) and choose a specific county in Romania using either typing or selecting from a list.
    • The app should support the ability to select and unselect multiple filters, allowing users to customize the displayed data on the charts.
    • Chart.js library should be integrated into the app to generate pie charts for education filters, horizontal bar charts for environment filters, and bar charts for age filters.
    • The app should retrieve unemployment data from the database using appropriate APIs based on the selected filters.
    • The generated charts should be interactive, allowing users to hover over chart elements to view labels and data.

4.2. Login

  • 4.2.1 Description and Priority
    • Description: This feature allows users to log in to the Unemployment Statistics App using their username and password. The app will search for the entered credentials in the database and validate them. If the credentials are correct, the user will be logged in and granted access to the app's features. If the credentials are incorrect, a pop-up notification will inform the user about the incorrect login attempt. Once logged in, users can navigate and interact with the charts. Additionally, an admin role exists, which grants access to user account management, including the ability to delete user accounts. The login feature is essential for user authentication and security.
    • Priority: High
  • 4.2.2 Stimulus/Response Sequences List
    • Feature: Login
      • Stimulus:
    • User enters their username and password and clicks the login button.
      • Response:
    • The app validates the entered credentials by searching for them in the database.
    • If the credentials are correct, the user is logged in and granted access to the app's features, including chart interaction.
    • If the credentials are incorrect, a pop-up notification is displayed, informing the user about the incorrect login attempt.
    • Feature: Account Management (Admin Role)
      • Stimulus:
    • Admin user logs in.
      • Response:
    • The app recognizes the user as an admin and provides additional options for account management.
    • The admin user can view a list of user accounts and choose to delete specific user accounts if necessary.
    • Feature: Logout
      • Stimulus:
    • User clicks on the logout button.
      • Response:
    • The app logs the user out, terminating their session and returning to the login screen.
  • 4.2.3 Functional Requirements
    • The app should provide a login interface where users can enter their username and password.
    • The app should provide a logout button or option to allow users to log out of their session.
    • The admin user should be able to view a list of user accounts and delete specific user accounts if necessary.

4.3 Register

  • 4.3.1 Description and Priority
    • Description: This feature allows users to register for a new account in the Unemployment Statistics App. Users can access the registration page from the login page if they do not have an existing account. During the registration process, users enter their email, choose a password, and confirm the password. Once registered, users can then log in to the app using their newly created account credentials. The register feature is essential for allowing new users to join and access the app's functionalities.
    • Priority: High
  • 4.3.2 Stimulus/Response Sequences List
      • Stimulus:
    • User clicks on the register button on the login page.
    • User successfully completes the registration form.
      • Response:
    • The app redirects the user to the registration page.
    • User enters their email, chooses a password, and confirms the password.
    • User clicks the register button to submit the registration form.
    • The app validates the user's input and creates a new account using the provided email and password.
    • The newly created account is stored in the database.
    • The app confirms the successful account creation and redirects the user back to the login page.
  • 4.3.3 Functional Requirements
    • The app should provide a registration page accessible from the login page.
    • The registration page should include input fields for the user's email, password, and password confirmation.
    • The app should validate the user's input, ensuring that the email is in the correct format and that the password and confirmation match.

4.4 Map

Users can apply filters for the time frame and general unemployment information. The map visually represents the data using different color ranges, indicating the number of unemployed individuals in each county based on the selected filters. The Map feature is a key component of the app, allowing users to visually explore and analyze unemployment data geographically.
    • Priority: High
  • 4.4.2 Stimulus/Response Sequences List
      • Stimulus:

      • Response:

    • The app generates a map of Romania with different colors representing the number of unemployed individuals in each county.
    • User can interact with the map, zoom in/out, and hover over counties to view detailed information about the unemployment rate and other relevant data.
  • 4.4.3 Functional Requirements

    • The app should generate an interactive map of Romania, displaying different colors representing the number of unemployed individuals in each county.
